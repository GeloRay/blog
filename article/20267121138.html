记llama.cpp的编译

llama.cpp是一个推理框架，支持CPU和vulkan后端，而且（据说）性能比ollama更好。

最近一时兴起，想在自己的电脑上跑起来Qwen3.6的35B版本，左看右看，ollama的vulkan后端不太稳定,而上vllm又有点杀鸡用牛刀的感觉（况且这玩意不支持图形API作后端），就选定了llama.cpp这个能支持核显的老古董作框架。考虑到软件源里可能没有这玩意，决定手动编译。

起·准备依赖

以下操作在Linux环境下进行。

为了提升性能，我需要编译带有vulkan后端支持的llama.cpp。整体步骤还是很简单的。首先执行vulkaninfo看是否有输出，有输出的继续，无输出或报错的装驱动。

之后在LunarG下载vulkan sdk安装，一路cd过去然后tar -xf解压，解压之后cd进去，执行

source setup-env.sh

在不关闭终端且不另起终端的条件下继续操作。

想关终端也不是不行，设置好PATH、LD_LIBRARY_PATH、VK_LAYER_PATH、VK_ADD_LAYER_PATH、PKG_CONFIG_PATH和CMAKE_PREFIX_PATH这几个环境变量就行。怎么配参考文档。

接下来是无趣的装依赖环节：

sudo dnf install  gcc gcc-c++ cmake spirv-headers-devel glslc xinput libXinerama xcb-util-cursor -y

承·编译

克隆仓库，没什么好说的（现在的网络环境下，个人更建议使用ssh克隆，速度又快又稳定）。

git clone https://github.com/ggml-org/llama.cpp

也没什么好说的：

cd llama.cpp
cmake -B build -DGGML_VULKAN=1
cmake --build build --config Release

编译需要大约10min（4线程），可以先去喝杯茶。

编译过程会报Warning，不用管。

按这样编译的版本没有https支持，如果需要https，安装openssl-devel。如果需要加速编译，安装ccache或添加-j（我上了-j 4）。编译产物在./build/bin下。

转·拉取模型

我打算使用Qwen3.6-35B-A3B，首先在魔搭上拉模型（拉之前需要确保git-lfs正确安装）：

git clone https://www.modelscope.cn/Qwen/Qwen3.6-35B-A3B.git

拉取的速度很慢，可以先去喝杯茶。

拉取后得到的safetensor文件是模型的本体，使用llama.cpp提供的convert_hf_to_gguf.py转成gguf文件后使用。缺依赖就跑pip install（这里有一个小坑：需要提前安装python-devel，否则numpy会缺依赖）。

合·推理

推理是很简单的

llama-cli -m /path/to/model -p PROMPT -ngl N

模型必须是gguf格式。-ngl参数指定在GPU上运行的层（ layer ）数（需要后端支持，默认为auto）。

在我的设备上，推理速度达到了惊人的0.1token/s，几乎不可用。模型有一部分进了swap，可能是一个原因。推理过程占用RAM约12GB。
